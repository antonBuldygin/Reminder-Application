Description

The looks are not enough. You need to add functionality too.

Now that you have a application window, it is a good time to learn how to provide functionality to those menus and buttons.

Since in this project the dropdown menu contains different inputs for the application, it is required that any change in its selection is noted by the application.

For the dropdown menu, we would need an item listener as the dropdown menu options are being selected and de-selected. When any item is selected then an item-selection event occurs and the itemStateChanged() method is invoked.

Therefore you need to implement your own logic for itemStateChanged() method.

Hint : In this method you must be able to identify which reminder is being set and at which intervals must the user be reminded.

 

Along with that, we have added a button in our application and right now it is unable to do anything when its clicked. 

To make the button come alive, an action listener must be added to it. An action listener can be added by using the addActionListener() method to each button that is declared in our application. Using action listeners you can handle events such as what must happen when the end user clicks on a button in the application window.

Note : ActionListener is an interface and addActionListener() allows you to bind the interface to your individual button directly. More info here.

Now that button click event is handled, you need to define what must happen when a certain button is clicked. To define that you need to implement logic for the actionPerformed() method which comes in ActionListener interface.

In this stage, after clicking the button a new screen must appear on your window with message related to the reminder that has been set.

So if you have a reminder for drinking water every 4 minutes, then your new screen should say something like "Its time to drink water".

This screen should have only one button on it that is capable of stopping the reminder. So set a button for this screen that would stop reminder and let you get back to the original screen of the application. Do not worry about the reminder timer logic at this stage, just implement the logic that would give back the original screen.

It is now time to add listeners to your menu options and buttons , so that you can make your reminder app do some real work.

You will also need to set up a new window that will appear when a reminder has been set. The new window must overwrite the previous content and refresh to new content.

Note : You must be able to revert to original opening window when needed so form your logic with that in mind.

 

Objectives

Add addActionListener()  to the button "ADD" with the name Add Reminder . Clicking on this button a new screen  with the name Set Reminder must appear. The title of this window should be Set Reminder . 

The window should contain the following components:

  • JTextField with the Field  name;
  • JLabel "Reminder Text" with the Reminder Text Label name. This label should be located over Field  JTextField ;
  • JButton "OK" with the  OKname;
  • JButton "Cancel" with the Cancelname;
  • JComboBox with the set Period name; 
  • JLabel with the Period label name.  This label should located beneth set Period  JComboBox ; 
  • JLabel "Set Period" with the Set Repeat    name. This label should located over set Period  JComboBox ;

 

  • JComboBox with the set Delay name; 
  • JLabel "Set Delay" with the Set Delay Label name. This label should located over set Delay  JComboBox ;
  • JLabel  with the Delays Label name. This label should located beneth set Delay  JComboBox .
